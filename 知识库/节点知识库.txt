# ShaderGraph 节点参考

## 约束
- 节点类型必须来自下表。
- 槽名称必须与表内一致（大小写敏感）。
- 输出 JSON 时仅使用这些预定义名称，不要使用数字 ID。
- 槽路径格式：`节点类型.输入输出类型.槽名称`（如 `AddNode.Input.A`）。

## 管线与块
- **Pipeline URP** 可选 Target：`Unlit`, `Lit`, `Fullscreen`, `Canvas`, `SixWay`, `Decal`, `SpriteLit`, `SpriteUnlit`, `SpriteCustomLit`
- **Pipeline BuiltIn** 可选 Target：`Unlit`, `Lit`, `Canvas`
- **Vertex Context 块**：`Position`, `Normal`, `Tangent`
- **Fragment Context 块**：`BaseColor`, `NormalTS`, `NormalOS`, `NormalWS`, `Metallic`, `Specular`, `Smoothness`, `Occlusion`, `Emission`, `Alpha`, `AlphaClipThreshold`, `CoatMask`, `CoatSmoothness`, `MapRightTopBack`, `MapLeftBottomFront`, `AbsorptionStrength`, `SpriteMask`
## Fragment Context 块可用性说明

不同 URP Target 下允许使用的 Fragment 块不同，且部分块受 **Surface Options** 开关控制。下方列出各 Target 的完整支持块集：

- **粗体** 表示创建时默认激活的块。
- *斜体* 表示需满足条件才会显示（如开启 Alpha Clipping、启用特定功能）。
- 未列出的块在该 Target 下**不可用**（即无法在 Master Stack 中添加）。

### URP Unlit Target
| 可用块 | 激活条件 |
|--------|----------|
| **BaseColor** | 默认激活 |
| *AlphaClipThreshold* | 开启 Alpha Clipping 后显示 |

> **注意**：Unlit 目标**不包含** `Alpha` 块（即使开启 Alpha Clipping 也不会显示），因为 Unlit 不向渲染管线输出 Alpha 通道。

### URP Lit Target
| 可用块 | 激活条件 |
|--------|----------|
| **BaseColor** | 默认激活 |
| **NormalTS** / **NormalOS** / **NormalWS** | 默认激活 `NormalTS`，其余可通过 “Fragment Normal Space” 切换 |
| **Metallic** (或 **Specular**) | 根据 Workflow Mode 默认激活其一；若开启 Material Override 则两者均显示 |
| **Smoothness** | 默认激活 |
| **Emission** | 默认激活 |
| **Occlusion** | 默认激活 |
| *Alpha* | 表面类型为 Transparent 或开启 Alpha Clipping 时显示（受 Material Override 影响） |
| *AlphaClipThreshold* | 开启 Alpha Clipping 时显示（受 Material Override 影响） |
| *CoatMask* | 勾选 “Clear Coat” 后显示 |
| *CoatSmoothness* | 勾选 “Clear Coat” 后显示 |

### URP Fullscreen Target
| 可用块 | 激活条件 |
|--------|----------|
| **BaseColor** | 默认激活 |
| **Alpha** | 默认激活 |

### URP Canvas Target
| 可用块 | 激活条件 |
|--------|----------|
| **BaseColor** | 默认激活 |
| **Emission** | 默认激活 |
| **Alpha** | 默认激活 |
| *AlphaClipThreshold* | 开启 Alpha Clipping 后显示 |

### URP SixWay Target
| 可用块 | 激活条件 |
|--------|----------|
| **BaseColor** | 默认激活 |
| **MapRightTopBack** | 默认激活 |
| **MapLeftBottomFront** | 默认激活 |
| *AbsorptionStrength* | 勾选 “Use Color Absorption” 或开启 Material Override 后显示 |
| **Emission** | 默认激活 |
| **Occlusion** | 默认激活 |
| *Alpha* | 表面类型为 Transparent 或开启 Alpha Clipping 时显示（受 Material Override 影响） |
| *AlphaClipThreshold* | 开启 Alpha Clipping 时显示（受 Material Override 影响） |

### URP Decal Target
| 可用块 | 激活条件 |
|--------|----------|
| **BaseColor** | 需在 SubTarget 面板中勾选 “Affect BaseColor” |
| **NormalTS** | 需勾选 “Affect Normal” |
| *NormalAlpha* (内部块) | 勾选 “Affect Normal” 且勾选 “Blend” 时启用 |
| **Metallic** | 需勾选 “Affect MAOS” |
| **Occlusion** | 需勾选 “Affect MAOS” |
| **Smoothness** | 需勾选 “Affect MAOS” |
| *MAOSAlpha* (内部块) | 勾选 “Affect MAOS” 时启用 |
| **Emission** | 需勾选 “Affect Emission” |
| **Alpha** | 勾选 “Affect BaseColor” 或 “Affect Emission” 时显示 |

### URP Sprite Lit / Sprite Unlit / Sprite CustomLit Target
| 可用块 | 激活条件 |
|--------|----------|
| **BaseColor** | 默认激活 |
| **NormalTS** | 仅 `SpriteLit` 和 `SpriteCustomLit` 默认激活 |
| **Emission** | 默认激活（`SpriteUnlit` 除外） |
| **Alpha** | 默认激活 |
| *AlphaClipThreshold* | 开启 Alpha Clipping 时显示 |
| **SpriteMask** | 所有 Sprite 目标均可用（默认不激活，需手动添加） |

## Built-in Pipeline Fragment Context 块可用性说明

Built‑in 管线支持以下 Target：`Lit`, `Unlit`, `Canvas`。各 Target 允许的 Fragment 块不同，且受 Surface Options 影响。

### Built‑in Lit Target
| 可用块 | 激活条件 |
|--------|----------|
| **BaseColor** | 默认激活 |
| **NormalTS** / **NormalOS** / **NormalWS** | 默认激活 `NormalTS`，可通过 “Fragment Normal Space” 切换 |
| **Metallic** | 默认激活（Workflow 固定为 Metallic，Specular 暂不支持） |
| **Smoothness** | 默认激活 |
| **Emission** | 默认激活 |
| **Occlusion** | 默认激活 |
| *Alpha* | 表面类型为 Transparent 或开启 Alpha Clipping 时显示（受 Material Override 影响） |
| *AlphaClipThreshold* | 开启 Alpha Clipping 时显示（受 Material Override 影响） |

> **注意**：Built‑in Lit 当前不支持 Specular 工作流，且 `CoatMask` / `CoatSmoothness` 不可用。

### Built‑in Unlit Target
| 可用块 | 激活条件 |
|--------|----------|
| **BaseColor** | 默认激活 |
| *Alpha* | 表面类型为 Transparent 或开启 Alpha Clipping 时显示（受 Material Override 影响） |
| *AlphaClipThreshold* | 开启 Alpha Clipping 时显示（受 Material Override 影响） |

> **注意**：Built‑in Unlit 不包含 `Emission` 块，且 `Alpha` 块仅在特定条件下出现（同 URP Unlit 行为）。

### Built‑in Canvas Target
| 可用块 | 激活条件 |
|--------|----------|
| **BaseColor** | 默认激活 |
| **Emission** | 默认激活 |
| **Alpha** | 默认激活 |
| *AlphaClipThreshold* | 开启 Alpha Clipping 后显示 |

> **说明**：Canvas Target 专用于 UI 着色器，渲染状态（如 Blend、Cull、Stencil）由 UI 系统统一控制，无需额外块。

## 条件性块触发规则（Built‑in 通用）

- **Alpha** 与 **AlphaClipThreshold**  
  受 `Surface Options` 中的 `Surface Type`（Opaque/Transparent）和 `Alpha Clipping` 开关控制。若开启 `Allow Material Override`，这两个块将始终显示并由材质属性驱动。

- **Material Override**  
  在 Lit/Unlit Target 属性面板中，若勾选 `Allow Material Override`，则原本受目标固定设置控制的块（如 `Alpha`、`Metallic`）会全部显示，并依赖材质属性动态启用。

- **Normal 空间切换（仅 Lit）**  
  可通过 `Fragment Normal Space` 下拉菜单在 `Tangent` / `Object` / `World` 之间切换，对应的 `NormalTS`、`NormalOS`、`NormalWS` 块会互斥显示。

## 条件性块触发规则（通用）

- **Alpha** 与 **AlphaClipThreshold**  
  是否可用取决于 `Surface Options` 中的 **Surface Type**（Opaque/Transparent）和 **Alpha Clipping** 开关。在 Lit/Unlit/SixWay 目标中，若开启 `Allow Material Override`，这两个块将始终显示（由材质控制）。

- **Material Override**  
  在 Lit/Unlit/SixWay Target 的属性面板中，若勾选 `Allow Material Override`，则原本受目标固定设置控制的块（如 `Alpha`、`Metallic`/`Specular`）将全部显示，并依赖材质属性动态启用。

- **Decal 块的可见性**  
  完全由 SubTarget 面板中的功能开关（Affect BaseColor、Affect Normal 等）决定，与 Master Stack 编辑无关。

## JSON 构建基本要求
- 必须包含 `"Pipeline"` 和 `"Target"` 字段。
- `"Nodes"` 数组中每个节点必须有 `"Id"` 和 `"Type"`。
- 连接关系在 `"Connections"` 中定义，使用 `"From"`, `"FromSlot"`, `"To"`, `"ToSlot"`。
- 输出到 Master 堆栈时，使用 `"Blocks"` 数组替代 MasterNode 连接。
- 槽路径格式：`节点类型.输入输出类型.槽名称`。

## CustomFunctionNode 构建规则（必读）
1. `functionName` 字段填写**不带精度后缀的基础名称**，例如 `"MyFunction"`。
2. `functionBody` 字段中的函数定义**必须包含精度后缀 `_float` 或 `_half`**，例如 `"void MyFunction_float(...)"`。
3. 多个输出插槽在 `Slots` 数组中以 `IsInput: false` 声明，每个输出可指定不同的 `ValueType`。
4. `DefaultValue` 始终为长度为 4 的数组 `[x, y, z, w]`。对于低维类型只使用前几个分量，例如 Vector1 使用 `[value, 0, 0, 0]`。

## CustomFunctionNode 常见问题与解决方案

### 1. 使用 URP 光照 API 时出现 `unrecognized identifier 'Light'` 或 `GetMainLight` 未定义
**原因**：预览模式下 URP 的 `Lighting.hlsl` 未被包含，且 ShaderGraph 主着色器包含顺序导致类型未定义。

**解决方案**：在 `functionBody` 中使用条件编译隔离预览模式，并显式包含头文件（仅非预览模式下）。同时传入世界位置以计算阴影坐标，兼容不同 URP 版本。

**正确示例（URP Unlit 卡通光照）**：
```json
{
  "Id": "CustomLightingFunc",
  "Type": "CustomFunctionNode",
  "Inputs": {
    "sourceType": "File",
    "functionName": "ToonLighting",
    "functionSource": "",
    "functionBody": "#if !defined(SHADERGRAPH_PREVIEW)\n#include \"Packages/com.unity.render-pipelines.universal/ShaderLibrary/Lighting.hlsl\"\n#endif\n\nvoid ToonLighting_float(\n    float3 WorldPos,\n    float3 WorldNormal,\n    float3 ViewDir,\n    float3 MainLightDir,\n    float3 BaseColor,\n    float3 RampColor,\n    float Roughness,\n    float AO,\n    float3 SpecularColor,\n    float SpecularSize,\n    float SpecularIntensity,\n    float RimPower,\n    float RimIntensity,\n    float3 RimColor,\n    out float3 OutColor)\n{\n    #if defined(SHADERGRAPH_PREVIEW)\n    OutColor = BaseColor;\n    #else\n    float4 shadowCoord = TransformWorldToShadowCoord(WorldPos);\n    Light mainLight = GetMainLight(shadowCoord);\n    float3 lightColor = mainLight.color;\n    float shadowAtten = mainLight.shadowAttenuation;\n\n    float NdotL = dot(WorldNormal, MainLightDir);\n    float halfLambert = NdotL * 0.5 + 0.5;\n    float3 diffuse = BaseColor * RampColor * lightColor * shadowAtten * AO;\n\n    float3 halfVec = normalize(ViewDir + MainLightDir);\n    float NdotH = saturate(dot(WorldNormal, halfVec));\n    float specMask = step(SpecularSize, NdotH) * (1.0 - Roughness);\n    float3 specular = specMask * SpecularIntensity * SpecularColor;\n\n    float fresnel = 1.0 - saturate(dot(WorldNormal, ViewDir));\n    float rim = pow(fresnel, RimPower) * RimIntensity;\n    float3 rimColor = rim * RimColor;\n\n    OutColor = diffuse + specular + rimColor;\n    #endif\n}",
    "usePragmas": true
  },
  "Slots": [ ... ]
}
2. 编译时提示 undeclared identifier 'MyFunction_float'
原因：functionName 与 functionBody 中的函数名精度后缀不一致。

规则：

functionName 填写不带精度后缀的基础名称（如 "ToonLighting"）。

functionBody 中的函数定义必须包含精度后缀 _float 或 _half（如 void ToonLighting_float(...)）。

3. 材质显示为黑色或只有环境光反射
原因：

未正确获取主光源数据（如 URP 版本需传入阴影坐标）。

在 Lit 目标下将自定义光照输出连接到了 BaseColor，导致与内置 PBR 叠加错误。

解决方案：

使用 Unlit 目标，将自定义光照最终颜色直接输出到 BaseColor 块。

确保场景中存在主光源，且强度 > 0。

在 functionBody 中使用 TransformWorldToShadowCoord(WorldPos) 获取阴影坐标。

4. 预览窗口正常，但场景中模型变黑
原因：预览模式与真实编译分支代码不一致，或真实编译时主光源数据获取失败。

调试技巧：

临时将 OutColor 设为 MainLightDir * 0.5 + 0.5 以验证方向向量传递是否正确。

检查 WorldPos 是否传入 AbsoluteWorld 空间位置。

在 Graph Settings 中勾选 "Preserve Specular Lighting" 和 "Preserve Lightmap UVs"（如果有）。
